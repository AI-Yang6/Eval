## 1. 快速开始

### 1.1 首次使用流程

```
第1步 → 创建一个测试集（手写或导入测试用例）
第2步 → 写一个 Prompt，保存为 v1
第3步 → 配置至少一个模型的 API Key
第4步 → 运行评估：选测试集 + Prompt + 模型，点"运行"
第5步 → 查看对比报告
```

### 1.2 首页概览

打开应用后进入 Dashboard，看到：
- **4 个 KPI 卡片**：测试集数量、Prompt 数量、启用模型数、评估总数
- **当前最优组合**：所有评估中得分最高的 Prompt 版本 × 模型组合
- **最近评估**：最近 5 次评估的列表，点击可跳转到对应报告
- **快捷操作**：底部 4 个快速入口

> 首次使用会看到空状态引导页，展示 4 步快速入门引导。

## 2. 测试集管理

测试集是评估的基础——它定义了你用来测试 AI 的"考题"集合。

### 2.1 创建测试集

1. 进入 **测试集** 页面
2. 点击 **新建测试集**
3. 填写名称和描述，点击创建

### 2.2 添加测试用例

在测试集详情页，你可以：
- **逐条添加**：点击"添加用例"，填写输入内容（必填）和期望输出（可选），可添加标签
- **批量导入**：点击"导入"，上传 JSON 或 CSV 文件
  - JSON 格式：`[{ "input": "...", "expected": "...", "tags": ["tag1"] }]`
  - CSV 格式：第一行为表头 `input,expected,tags`
- **编辑/删除**：点击用例行右侧的操作按钮

### 2.3 标签筛选

在测试集详情页顶部的标签栏点击标签，列表会自动筛选出包含该标签的用例。点击"全部"恢复显示所有用例。

### 2.4 分页

当测试用例超过 20 条时，会自动分页显示。底部有页码导航，可点击切换页面。

### 2.5 删除测试集

在测试集详情页右上角点击"删除测试集"按钮，确认后删除（不可恢复）。

## 3. Prompt 版本管理

### 3.1 创建 Prompt

1. 进入 **Prompt** 页面
2. 点击 **新建 Prompt**
3. 填写名称、System Prompt（系统指令）和 User Prompt 模板

### 3.2 版本管理

- **保存为新版本**：编辑 Prompt 内容后，点击"保存为新版本"，版本号自动递增
- **版本切换**：点击版本号标签（如 `v1`、`v2`）切换查看不同版本
- **版本对比**：在版本列表中选择两个版本，点击"对比"，可查看文本 diff
- **删除版本**：在版本列表中 hover 出现删除按钮（至少保留一个版本）

### 3.3 关联评估信息

每个版本标签旁会显示该版本的评估次数和最高得分（如 `v3 ✓ · 2次 · 4.10`），帮助你快速了解哪个版本表现最好。

### 3.4 变量支持

Prompt 模板支持 `{{变量名}}` 语法，运行时自动替换：
- `{{input}}` → 自动替换为当前测试用例的输入内容
- `{{variable}}` → 运行评估时需手动填写变量值

### 3.5 运行评估

在 Prompt 详情页点击"运行评估"，自动跳转到新建评估页面并预选当前 Prompt 版本。

## 4. 模型配置

### 4.1 配置 Provider

1. 进入 **模型** 页面
2. 点击 Provider 卡片上的"配置"按钮
3. 填入 API Key 和 Base URL（可选，有默认值）
4. 点击保存

### 4.2 启用/禁用模型

在 Provider 卡片内，勾选你想使用的具体模型（如 gpt-4o、claude-sonnet-4-6 等）。
只有启用的模型才会出现在评估选择列表中。

### 4.3 添加自定义模型

在 Provider 卡片中点击"添加自定义模型"，输入模型名称即可。
如果选择"自定义 Provider"类型，还需填写 Base URL 和 API Key。

### 4.4 支持的 Provider

| Provider | 默认模型 | 说明 |
|---|---|---|
| OpenAI | gpt-4o, gpt-4o-mini, o3-mini | 标准 OpenAI API |
| Anthropic | claude-sonnet-4-6, claude-haiku-4-5 | 标准 Anthropic API |
| DeepSeek | deepseek-chat, deepseek-reasoner | DeepSeek API |
| 通义千问 | qwen-plus, qwen-max | 阿里云通义千问 |
| GLM | glm-4-plus, glm-4-flash | 智谱 GLM |
| Kimi | moonshot-v1-8k | Moonshot |
| 豆包 | doubao-pro, doubao-lite | 字节跳动 |
| MiniMax | minimax-text-01 | MiniMax |
| 自定义 | 用户定义 | OpenAI 兼容接口 |

### 4.5 Embedding 统一配置

在模型页面底部有 **"Embedding 统一配置"** 卡片，用于知识库（RAG）的向量化：
- 配置一个独立的 API Key 和 Base URL（如通义千问的 Embedding 模型）
- 如果不配置，知识库会使用对应 Provider 的模型配置
- 推荐使用通义千问 `qwen3-embedding-4b` 等专用 Embedding 模型

## 5. 运行评估

### 5.1 创建评估任务

1. 进入 **新建评估** 页面
2. 按顺序填写以下内容：

#### 步骤 A：选择测试集
- 选择一个已有的测试集，右侧显示该测试集的用例数量

#### 步骤 B：选择 Prompt 版本
- 勾选一个或多个 Prompt 版本（可跨不同 Prompt）
- 选中的版本会出现在右侧已选列表

#### 步骤 C：选择模型
- 勾选一个或多个已启用的模型
- 选中的模型会出现在右侧已选列表

#### 步骤 D（可选）：关联知识库
- 如果想让 Prompt 运行时能检索知识库内容，选择一个知识库
- 设置检索数量（Top-K，默认 3）

#### 步骤 E：配置评估维度
- 选择评分模板（通用对话、客服质量、内容创作、代码生成、翻译质量）
- 或自定义评估维度：填写维度名称、描述和权重
- 维度数量建议 2-5 个

#### 步骤 F：选择 Judge 模型
- 选择用于打分的模型（建议选择性价比高的模型如 claude-haiku-4-5 或 gpt-4o-mini）

#### 步骤 G：填写评估名称
- 自动生成名称，可手动修改

### 5.2 执行过程

1. 点击 **开始评估** 后，跳转到评估详情页
2. 页面会实时展示每个组合（Prompt × 模型）的执行进度
3. 系统自动并发执行（默认 4 个并发），避免触发 API Rate Limit

### 5.3 中止与重试

- **中止**：评估运行时，可点击"中止"停止剩余未执行的任务
- **重试失败项**：如果有失败的用例，可点击"重试失败"重新执行
- **删除评估**：评估完成后可删除整个评估记录

### 5.4 状态说明

| 状态 | 含义 |
|---|---|
| 运行中 | 正在执行 API 调用 |
| 已完成 | 所有用例执行完毕 |
| 已中止 | 用户手动中止 |
| 失败 | 执行过程中出现错误 |

## 6. 查看评估报告

报告是产品的核心页面，展示评估的完整结果。

### 6.1 报告总览

报告页顶部展示：
- **综合得分**：大数字展示整体评分（≥4.0 绿色、3.0-3.9 紫色、<3.0 橙色）
- **最优组合**：得分最高的 Prompt 版本 × 模型组合
- **基本信息**：测试集名称、用例数、评估时间

### 6.2 雷达图

- 展示所有组合在各个评估维度上的得分对比
- 每个组合用不同颜色的多边形表示
- 维度越往外延伸代表得分越高

### 6.3 各组合详情

每个组合（Prompt 版本 × 模型）单独展示：
- 综合得分
- 各维度得分
- 该组合下每个测试用例的具体评分

### 6.4 人工打分

在报告页中，可对每个测试用例的输出进行人工打分（1-5 分）：
1. 展开某个组合的用例列表
2. 在评分列点击"打分"按钮
3. 选择 1-5 分并填写评语（可选）
4. 提交后自动更新该组合的平均分

> 人工打分后，报告页会显示 **Judge 校准度分析**，对比 LLM 打分与人工打分的偏差。

### 6.5 Bad Case 标记

在用例详情中，可点击"标记为 Bad Case"：
- 标记后的用例会在报告中显示为红色高亮
- 下次评估时会自动检测这些 Bad Case 是否修复

### 6.6 Bad Case 回归检测

当查看已完成评估的报告时，系统会自动检测：
- **已修复**：之前标记的 Bad Case 在新版本中得分提升 ≥0.5 分 → 绿色标签
- **仍异常**：之前标记的 Bad Case 在新版本中仍未改善 → 红色标签
- 点击每条记录可查看具体变化

### 6.7 导出报告

在报告页支持导出：
- **CSV**：下载所有评估结果的表格数据
- **Markdown**：下载格式化的报告文档
- **JSON**：下载完整原始数据

## 7. 评估历史与趋势

### 7.1 历史列表

进入 **历史** 页面，查看所有历史评估记录：
- 按时间倒序排列
- 展示综合得分、Prompt 名称、模型、状态
- 点击"查看"进入对应报告

### 7.2 综合得分趋势图

当有 2 次及以上评估记录时，自动生成折线图：
- X 轴：时间
- Y 轴：综合得分（0-5 分）
- 按 Prompt × 模型组合分组显示不同颜色线条
- 直观展示优化方向是否正确

### 7.3 筛选

可按 Prompt 名称和模型标签筛选历史记录，趋势图会同步更新。

### 7.4 删除评估

在历史列表中，通过下拉菜单可删除评估记录（不可恢复）。

## 8. 知识库（RAG）系统

知识库让评估时能检索外部文档内容，适用于需要"开卷考试"的场景。

### 8.1 创建知识库

1. 进入 **知识库** 页面
2. 点击 **新建知识库**
3. 填写名称、描述（可选）
4. 选择 Embedding 模型（如 qwen3-embedding-4b）
5. 点击创建

### 8.2 上传文档

在知识库详情页：
1. 点击 **上传文档**
2. 上传 .txt / .md / .jsonl 文件，或直接粘贴文本内容
3. 系统自动切片并向量化
4. 上传完成后显示片段数量

### 8.3 关联评估

在新建评估时，可以勾选关联一个知识库。评估执行时，每个测试用例会自动检索知识库中最相关的内容，拼接在 Prompt 中发给模型。

### 8.4 删除知识库/文档

- 删除文档：在文档列表中 hover 出现删除按钮
- 删除知识库：在知识库详情页右上角点击"删除知识库"（会同时删除所有文档和向量片段）

## 9. 数据备份与恢复

### 9.1 导出备份

1. 点击侧边栏底部的 **备份** 按钮
2. 点击 **导出备份**
3. 浏览器自动下载一个 JSON 文件，包含所有数据（测试集、Prompt、模型配置、评估结果、知识库、Embedding 配置）

### 9.2 导入备份

1. 点击侧边栏底部的 **备份** 按钮
2. 点击 **导入备份**
3. 选择之前导出的 JSON 文件
4. 系统会自动恢复所有数据

> 注意：导入备份会覆盖当前所有数据。

### 9.3 存储说明

所有数据存储在浏览器的 IndexedDB 中：
- 清除浏览器数据会导致数据丢失
- 建议定期导出备份
- 不同浏览器/设备之间不共享数据

## 10. 常见问题

### Q1：数据存在哪里？会不会丢失？

所有数据存储在浏览器本地 IndexedDB 中。清理浏览器缓存或使用无痕模式可能会导致数据丢失。**建议定期导出备份**。

### Q2：API Key 安全吗？

API Key 随模型配置存储在浏览器 IndexedDB 中，仅在你本地使用。每次调用模型时，请求通过 Next.js API Route 代理转发到对应 Provider；不要在不可信环境中部署或共享带有个人 API Key 的浏览器数据。

### Q3：评估速度慢怎么办？

评估速度取决于：
- **API 响应速度**：不同模型响应时间不同
- **并发数**：系统默认 4 个并发，可在评估详情页观察进度
- **测试集大小**：测试用例越多，耗时越长

### Q4：Judge 打分准吗？

LLM-as-Judge 的评分仅供参考。建议：
- 在报告页进行**人工打分**，系统会自动分析 Judge 校准度（MAE、Pearson 相关系数、维度偏差）
- 定期检查 Judge 评分是否与你的判断一致
- 调低你不信任的维度的权重

### Q5：怎么对比两次评估？

有两种方式：
- **历史页趋势图**：自动展示所有评估的得分趋势
- **报告页回归检测**：自动对比本次评估与历史 Bad Case

### Q6：支持哪些 Embedding 模型？

推荐使用：
- 通义千问 `qwen3-embedding-4b`（免费额度高）
- OpenAI `text-embedding-3-small` / `text-embedding-3-large`
- 其他 OpenAI 兼容的 Embedding 模型

### Q7：自定义模型怎么配置？

选择"自定义 Provider"，填写：
- 名称（显示用）
- Base URL（API 端点）
- API Key
- 模型名称（如 `gpt-4o-mini`）

### Q8：为什么有些评估没有 Judge 校准度？

Judge 校准度需要同时有人工评分和 LLM 评分才能计算。如果你没有在报告页进行人工打分，校准度部分不会显示。
