## 1. 全局设计规范

### 1.1 设计原则
- **30 秒上手**：核心流程（创建测试集 → 跑评估 → 看报告）不需要教程
- **数据驱动审美**：报告页是门面，交互页是工具 — 工具够用就好，报告必须惊艳
- **渐进式复杂度**：必填项最少化，高级选项折叠

### 1.2 UI 框架
- 组件库：shadcn/ui（基于 Radix + Tailwind）
- 图标：Lucide
- 图表：recharts
- 布局：左侧导航 + 右侧内容区

### 1.3 全局布局
```
┌──────────┬───────────────────────────────┐
│          │  Header: 产品名 + 全局操作      │
│  侧边栏   ├───────────────────────────────┤
│          │                               │
│  测试集   │                               │
│  Prompt  │        主内容区                │
│  模型     │                               │
│  评估     │                               │
│  历史     │                               │
│          │                               │
└──────────┴───────────────────────────────┘
```

### 1.4 状态规范
- 空状态：每个列表页都有引导卡片（"创建你的第一个测试集"）
- 加载：骨架屏（Skeleton）
- 错误：toast 提示，不阻断操作
- 成功：轻量 toast，不打断流程

## 2. 测试集管理

### 2.1 用户故事
> 作为 AI PM，我想管理一组测试用例，这样我每次改 prompt 后都能用同一组 case 跑评估，确保对比公平。

### 2.2 页面：测试集列表页 `/test-suites`

**布局**：
```
┌─────────────────────────────────────────┐
│  测试集  [+ 新建测试集]  [导入 JSON/CSV]   │
├─────────────────────────────────────────┤
│                                         │
│  📦 客服场景-退款流程    30条  上次:2h前    │
│  📦 客服场景-问候语      15条  上次:1d前    │
│  📦 法律咨询-合同审查    22条  上次:3d前    │
│                                         │
│  (空状态: "还没有测试集，点击新建开始")      │
└─────────────────────────────────────────┘
```

**验收标准**：
- [ ] 展示所有测试集，按最近使用时间倒序
- [ ] 每条显示：名称、用例数量、最近使用时间
- [ ] 点击进入详情页
- [ ] 支持删除（二次确认）
- [ ] 空状态有引导

### 2.3 页面：测试集详情页 `/test-suites/[id]`

**布局**：
```
┌─────────────────────────────────────────┐
│  ← 返回  客服场景-退款流程    [编辑] [删除] │
│                                         │
│  描述：退款相关用户问题，期望引导至退款流程   │
│  类型：单轮对话                           │
├─────────────────────────────────────────┤
│  测试用例 (30条)        [+ 添加] [导入]    │
│  筛选标签: [退款] [流程] [政策]            │
├──────┬──────────────┬────────┬──────────┤
│  #   │  输入 (input) │  期望   │  标签     │
├──────┼──────────────┼────────┼──────────┤
│  1   │  我想退款     │ 引导退  │ 退款,流程 │
│  2   │  退款多久到账  │ 说明时  │ 退款,政策 │
│  ... │              │        │          │
└──────┴──────────────┴────────┴──────────┘
```

**验收标准**：
- [x] 展示测试集元信息（名称、描述、类型、创建时间）
- [x] 表格展示所有用例，支持分页（每页 20 条），底部页码导航
- [x] 可逐条添加 / 编辑 / 删除用例
- [x] 输入(input)必填，期望(expected)选填，标签(tags)选填
- [x] 支持标签筛选

### 2.4 新建 / 导入测试集

**新建流程**：
1. 弹出 Dialog：填写名称 + 描述 + 类型（单轮对话 / 对话序列）
2. 进入详情页，逐条添加用例

**导入流程**：
1. 点击"导入" → 弹出 Dialog → 选择文件或粘贴 JSON
2. JSON 格式：
```json
[
  {"input": "我想退款", "expected": "引导退款流程", "tags": ["退款"]},
  {"input": "退款多久到账", "expected": "说明时效", "tags": ["退款", "政策"]}
]
```
3. CSV 格式：第一行为表头 `input,expected,tags`，tags 用分号分隔
4. 预览解析结果 → 确认导入

**验收标准**：
- [ ] 支持 JSON 文件上传
- [ ] 支持 CSV 文件上传
- [ ] 支持直接粘贴 JSON 文本
- [ ] 解析失败时给出具体错误提示（第 N 行格式错误）
- [ ] 导入前可预览前 5 条

### 2.5 数据字段

```
TestSuite {
  id: string (uuid)
  name: string (required, max 100)
  description: string (optional, max 500)
  type: 'single-turn' | 'multi-turn'
  cases: TestCase[]
  createdAt: string (ISO 8601)
  updatedAt: string (ISO 8601)
}

TestCase {
  id: string (uuid)
  input: string (required)
  expected: string (optional)
  tags: string[] (optional)
}
```

- [ ] 展示所有 Prompt，显示版本数和最新版本号
- [ ] 点击进入详情页

### 3.3 页面：Prompt 详情页 `/prompts/[id]`

**布局**：
```
┌─────────────────────────────────────────┐
│  ← 返回  客服回复-prompt    [编辑当前版本]  │
├─────────────────────────────────────────┤

**验收标准**：
- [x] 展示当前版本的 system prompt + user prompt 模板

- [x] "用此版本运行评估"：跳转到评估创建页，自动选中该 prompt 版本
- [x] 版本标签上显示关联评估次数；版本详情底部显示最高分

### 3.4 新建 Prompt

1. 弹出 Dialog：填写名称
2. 进入详情页的编辑模式
3. 填写 system prompt + user prompt 模板
4. 保存 → 自动生成 v1

### 3.5 数据字段

```
Prompt {
  id: string (uuid)
  name: string (required, max 100)
  versions: PromptVersion[]
  createdAt: string (ISO 8601)
}

PromptVersion {
  id: string (uuid)
  versionNumber: number (auto-increment)
  systemPrompt: string (required)
  userPromptTemplate: string (default: "{{input}}")
  createdAt: string (ISO 8601)
}
```

## 4. 模型配置

### 4.1 用户故事
> 作为 AI PM，我想配置多个 AI 模型，这样我可以在评估时并行对比不同模型在同一任务上的表现。

### 4.2 页面：模型配置页 `/models`

**布局**：
```
┌─────────────────────────────────────────┐
│  模型配置  [+ 添加模型]                    │
├─────────────────────────────────────────┤
│                                         │
│  ┌─ OpenAI ────────────────────────────┐│
│  │  ✅ GPT-4o          Key: sk-...3f2  ││
│  │  ✅ GPT-4o-mini     Key: sk-...3f2  ││
│  └─────────────────────────────────────┘│
│                                         │
│  ┌─ Anthropic ─────────────────────────┐│
│  │  ✅ Claude Sonnet 4.6  Key: sk-...8a1││
│  │  ✅ Claude Haiku 4.5   Key: sk-...8a1││
│  └─────────────────────────────────────┘│
│                                         │
└─────────────────────────────────────────┘
```

**验收标准**：
- [ ] 按 Provider 分组展示已配置模型
- [ ] 每个 Provider 填一次 API Key，该 Provider 下所有模型共用
- [ ] 支持"测试连接"按钮，验证 API Key 是否有效
- [ ] 支持删除模型配置

### 4.3 添加模型流程

1. 点击"添加模型" → 弹出 Dialog
2. 选择 Provider（下拉：OpenAI / Anthropic）
3. 填写 API Key（密码输入框，可切换显示）
4. 自动列出该 Provider 的可用模型，勾选要启用的
5. 点击"测试连接" → 验证 API Key
6. 保存

**验收标准**：
- [ ] API Key 输入框默认隐藏，可切换显示
- [ ] 测试连接成功/失败有明确反馈
- [ ] 同一 Provider 只需填一次 Key

### 4.4 数据字段

```
ModelConfig {
  id: string (uuid)
  provider: 'openai' | 'anthropic'
  providerLabel: string ("OpenAI" | "Anthropic")
  apiKey: string (加密存储于 localStorage)
  models: ModelDefinition[]
}

ModelDefinition {
  id: string (uuid)
  modelId: string (e.g. "gpt-4o", "claude-sonnet-4-6-20250514")
  label: string (e.g. "GPT-4o", "Claude Sonnet 4.6")

### 5.4 评估维度模板（预设）

为降低用户填写成本，提供以下预设维度模板：

**通用评估模板**：
| 维度 | 描述 |
|---|---|
| 准确性 | 回复内容是否事实准确、无幻觉 |
| 完整性 | 是否完整覆盖了用户问题的所有方面 |
| 清晰度 | 回复是否条理清晰、易于理解 |
| 相关性 | 回复是否紧扣用户问题，无无关内容 |

**客服场景模板**：
| 维度 | 描述 |
|---|---|
| 亲切度 | 回复语气是否亲切友好 |
| 专业度 | 是否使用了准确的专业术语和政策 |
| 解决力 | 是否有效解决了用户问题或提供了可行方案 |

**创意写作模板**：
| 维度 | 描述 |
|---|---|
| 创意性 | 内容是否新颖、有创意 |
| 吸引力 | 是否能吸引读者继续阅读 |
| 风格匹配 | 是否符合指定的写作风格 |

### 5.5 数据字段

```
EvalRun {
  id: string (uuid)
  name: string (auto: "{prompt名} 评估 {timestamp}")
  status: 'running' | 'completed' | 'failed'
  testSuiteId: string

  modelIds: string[]

  judgeModelId: string
  createdAt: string (ISO 8601)
  completedAt: string | null

RubricDimension {
  name: string (e.g. "亲切度")

  id: string (uuid)
  testCaseId: string
  promptVersionId: string
  modelId: string
  actualOutput: string
  tokenUsage: { input: number, output: number }
  latency: number (ms)
  scores: { [dimensionName]: number } // 1-5
  judgeReasoning: string
  judgeModelId: string
}

EvalError {
  testCaseId: string
  promptVersionId: string
  modelId: string
  error: string
}
```
